How It Works

Goal to lesson in three steps

No generic courses. Every lesson is shaped to the learner's specific gap, role, and timeline — generated once access is confirmed at the database level.

1
Tell us your goal
What you want to learn, where you're starting, and what you need to achieve by when.
2
AI builds your lesson
A personalised lesson blueprint is generated for your specific gap — not a generic course catalogue.
3
Track your progress
Cloud-synced across devices. Access enforced at database level — not just the frontend.
🔐 Firestore rules enforce access — no client-side bypass possible
Key Features

Built for real learners,
enforced at the database.

🤖

AI Lesson Blueprints

Every lesson is AI-generated to match the learner's stated goal, starting level, and target timeline — a personalised path, not a catalogue you scroll through.

🔐

Firestore Security Rules

Per-user access enforced at the database layer, not the frontend. A user can only read the content their tier and role permit — server-side, with no client-side bypass.

🖥️

Admin Console

Full interface for user management, role and tier assignment, module access toggles, course authoring, and access request fulfilment — all from one panel.

🔄

OTA Content Updates

Lesson HTML is served from Firebase Storage. Content can be added, updated, or swapped without a redeploy — content lifecycle is fully decoupled from platform code.

🌍

Multilingual — EN · AR · DE

Full i18next integration with RTL layout for Arabic. The platform interface and lesson metadata are fully translatable at runtime without a page reload.

Personalised Modules

AI-generated lessons, specific to each learner

Beyond the standard tracks, SkillMap generates bespoke lesson blueprints based on the learner's questionnaire: current role, target job, existing knowledge, and deadline. These personalised modules live in isolated Firestore collections, accessible only to the user they were generated for.

Admins review access requests, author lesson HTML in the admin console, and publish directly to the learner — no redeploy, no waiting.
