Core Features

What E·OVR Does

🔄

Controlled Incident Workflow

Strict 5-stage state machine: Created → InProgress → Evaluating → ActionTaken → Completed, with a MoreInfoNeeded loop. Every transition is validated server-side and appended to an immutable audit trail.

🤖

AI Classification Hook

Provider-adapter service supports OpenAI, Anthropic, and Google. On incident submission, the AI layer writes auto_classification, event_type, risk score, and signal flags — before the Mongo insert. Fully optional; failures never block submission.

📊

Scope-Gated Analytics

Aggregation pipeline serves status/severity breakdowns, monthly trend series, and facility comparison charts. Each endpoint enforces a minimum tier — tier 2 for summary, tier 3 for trends, tier 4 for cross-facility comparison.

🔐

CSFLE Field Encryption

MongoDB Client-Side Field Level Encryption protects medical_file_number with AES-256 deterministic encryption before it ever leaves the application. Works independently of server-side encryption; plaintext never transits the wire.

🔗

Anonymous Patient Reporting

Each facility gets a UUID-based public link. Patients submit incidents without any login. The UUID maps to a facility server-side; the patient's incident is created as reporter_type=patient and enters the same workflow.

🛡️

Dual-Token Auth + Silent Refresh

Short-lived access token in memory + long-lived refresh token in httpOnly cookie, hashed in DB. Axios interceptor silently refreshes on 401 and retries the original request. Logout invalidates the stored hash.

AI Classification

Provider-Agnostic AI Layer

The AI hook fires inside the incident creation service before the Mongo insert. If the provider call succeeds, structured metadata is written into ai_metadata: auto-classification, event type, confidence score, risk estimate, and signal flags.

Quality admins review suggestions via the AIBadge — accepting or overriding the classification. Feedback (AI suggestion vs. human choice) is stored as a labelled dataset for future fine-tuning.

Security Model

Defence in Depth

🔑
Dual-Token JWT

15-min access token in memory + 30-day refresh token in httpOnly cookie. Refresh tokens are hashed in DB; up to 10 retained per user. Logout invalidates hash.

🧬
CSFLE Encryption

medical_file_number encrypted client-side with AEAD-AES-256-CBC-HMAC-SHA-512 (deterministic). Local KMS master key; plaintext never reaches the server wire.

🛂
Role Gating

require_role(...) FastAPI dependency at every router. Scope filter builds Mongo queries dynamically — facility / administration / governorate / all — based on JWT claims.

🔗
Patient UUID Links

UUID v4 per facility grants anonymous submission only — no read access, no auth tokens. Incident is tagged reporter_type=patient and enters standard workflow.
